1. Introduction to JTML

JTML integrates markup-like syntax with powerful scripting capabilities. It excels as both a general-purpose language (with strong typing, classes, modules, concurrency, cryptography) and a web development scripting language (generating HTML elements with # syntax, interacting with databases, dynamic UI).

Every statement ends with \\. Blocks end with a standalone \\ line. This uniform approach simplifies parsing and ensures visually distinctive code structure.

2. Getting Started

2.1. Installation

pip install jtml-compiler
jtml --version

2.2. Hello World Example

#p: "Hello, World!" #p\\
\\
Run: 
jtml run hello_world.jtml
**Output:** 
<p>Hello, World!</p>

3. Language Fundamentals

JTML uses Unicode, supports string literals in quotes, and comments with // or /* ... */. Keywords are lowercase. Types include primitives and user-defined types.

3.1. Lexical Conventions

  • Identifiers: letters, digits, underscore.
  • Strings: double quotes with escape sequences.
  • Comments: // single-line, /* multi-line */.

3.2. Syntax Overview

All statements end with \\, blocks end with \\ line. Control structures and functions follow this pattern. No semicolons, ensuring a clean, visually distinct style.

3.3. Data Types

Primitives: int, float, string, bool, void. Composite: list, dict. User-defined classes as custom types.

3.4. Variables and Constants

define count = 0\\
const PI: float = 3.1415\\

3.5. Operators

Arithmetic (+, -, *, /), Comparison (==, !=, <=, >=), Logical (&&, ||, !), Assignment (=, +=, ...)

3.6. Expressions and Statements

define x = (y + 3) * 2\\
show x\\

4. Control Flow

Conditional statements, loops, and jumps allow structured, readable flow control.

4.1. Conditional Statements

if (x > 5) \\
    show "Greater"\\
else \\
    show "Not greater"\\
\\

4.2. Loops

while (count < 5) \\
    show count\\
    count += 1\\
\\

for (item in items) \\
    show item\\
\\

4.3. Jump Statements

break, continue, return, throw end with \\ and integrate seamlessly with control structures.

5. Functions

Functions define reusable logic. Async functions support asynchronous operations, ideal for web or I/O tasks.

5.1. Function Definitions

function add(a:int,b:int):int \\
    return a+b\\
\\

async function fetchData():void \\
    // asynchronous operations
    return\\
\\

5.2. Function Calls

define result = add(3,4)\\

5.3. Recursion

function factorial(n:int):int \\
    if (n <= 1) \\
        return 1\\
    else \\
        return n * factorial(n-1)\\
    \\
\\

6. Data Structures

Lists, arrays, dictionaries integrate seamlessly with the rest of JTML, supporting advanced operations and dynamic typing.

6.1. Arrays and Lists

define numbers: list[int] = [1,2,3]\\
numbers.append(4)\\

6.2. Dictionaries

define capitals: dict[string,string] = {
    "France":"Paris",
    "Italy":"Rome"
}\\
show capitals["France"]\\

7. Classes and Objects

Classes provide a blueprint for objects. Inheritance and constructors enable OOP design patterns, making JTML suitable for large-scale projects.

7.1. Class Definitions

class Person \\
    define name: string\\
    define age: int\\
    
    function constructor(name:string,age:int):void \\
        self.name = name\\
        self.age = age\\
    \\
\\

7.2. Object Creation

define alice = new Person("Alice",30)\\

7.3. Inheritance

class Employee derive Person \\
    define employeeId:int\\
    function constructor(name:string,age:int,employeeId:int):void \\
        super(name,age)\\
        self.employeeId = employeeId\\
    \\
\\

8. JTML Elements

JTMLâ€™s #tag syntax integrates markup. #-demanding keywords require a closing #. You may repeat the opening tag name before the closing hash for clarity. They are statements, so should in all cases end with the custom \\ escape sequence.

8.1. Element Syntax

#p "Hello, World!" #\\
\\

#p "Hello, World!" #p\\
\\

Both forms are valid. The second form (#p ... #p) improves readability.

8.2. Attributes and Properties

#div class:"container" id:"main"
    #p "Hello" #p\\
#div\\

8.3. Dynamic Content

define userName = "Alice"\\
#p "Welcome, #(userName)" #p\\

9. Advanced Features

JTML excels beyond simple scripting: integrate databases, quantum ops, and cryptography. Perfect for full-stack apps, research tools, or secure computations.

9.1. Database Interactions

connect to "sqlite:///mydb.db" as db\\
define users = query on db:"SELECT * FROM users"\\

Use parameterized queries, transactions, and integrate with the UI seamlessly.

9.2. Quantum Computing Simulation

define q1 as qubit\\
define q2 as qubit\\
apply H on q1\\
apply CNOT on q1,q2\\
define r = measure q1\\

Simulate quantum algorithms and integrate results into your web app or data pipeline.

9.3. Cryptography Operations

define privateKey = generate_key type:"RSA", size:2048\\
define publicKey = derive_public_key from privateKey\\
define enc = encrypt data: message with key:publicKey algorithm:"RSA"\\
define dec = decrypt data: enc with key:privateKey algorithm:"RSA"\\

Handle encryption, decryption, hashing, and signing to secure data end-to-end.

10. Error Handling

Use try/catch/finally and throw exceptions. JTML integrates error handling gracefully into its syntax:

try \\
    define res = 10/0\\
catch (DivisionByZeroException e) \\
    show e.message\\
finally \\
    show "Done"\\
\\

11. Modules and Packages

Modularize large projects by splitting code into multiple JTML files and importing them. Packages group related modules, enabling clean architecture:

import "utils/math.jtml"\\
define answer = add(40,2)\\
show answer\\

Build reusable libraries, share common code across projects, and keep codebases maintainable.

12. Input/Output Operations

JTML provides file handling primitives, perfect for web scripts generating static content or reading configuration files:

define data = read_file("input.txt")\\
data = data + " More info"\\
write_file("output.txt", data)\\

Combine with crypto or DB features for secure data processing pipelines.

Leverage multiple threads to run tasks in parallel:

t.start()\\

Build scalable servers, parallel computations, or asynchronous IO operations easily.

14. Annotations and Decorators

Annotations add metadata to classes, functions, or properties. Decorators wrap functionality, enabling aspects like logging, caching, or deprecation warnings.

@Deprecated("Use newFunc instead")
function oldFunc(): void \\
    // ...
    return\\
\\

Future expansions: custom decorators for validation, security checks, or performance monitoring.

15. Best Practices

  • Consistent Syntax: End every statement with \\, close blocks with \\ on a lone line.
  • Descriptive Names: Use meaningful identifiers for variables, functions, classes.
  • Modularity: Use modules and packages to organize code. Separate UI logic from core logic.
  • Security and Performance: Validate input, parameterize queries, handle exceptions gracefully, and leverage concurrency responsibly.
  • Testing and Documentation: Write unit tests, document complex logic, and maintain a reference guide like this one.
